In comparison, it will seem like we have to do a lot more to get a basic "Hello, World!" program up and running with Django. However, Django's value won't be evident from the initial stage of a project. Adding a database, user authentication, or any number of features can be done with less effort due to that functionality being built in.

Although Flask requires you to create these yourself, its simplicity means there's less overhead. You're free to implement only what you need rather than installing a large library filled with features you won't use.

Overview

Flask and Django are two popular frameworks for Python. Here's a brief feature comparison:

Django Flask
Full-stack (“batteries included”) Micro-framework - only the basics are included by default
Low flexibility High flexibility
Built-in ORM, supports multiple databases Supports multiple databases but relies on third-party modules for integration
Built-in bootstrapping (Django-admin) N/A
Built-in template engine Relies on jinja2
Model View Template (MVT) Model View Controller (MVC)

When should you use Flask? When should you use Django?

Use Django if you need any of the following: Use Flask if you need any of the following:
You're building a large or multi-page project You're building a small or single page project
You have a strict deadline You're building a static site or RESTful services
You don't have time to build various features from scratch or just want to utilize Django's built-in admin You need flexibility in choosing libraries
You want to use native ORM support You don't need a database or want to have flexibility in choosing one
You want to easily extend the project to be more complicated in the future You already have API support
